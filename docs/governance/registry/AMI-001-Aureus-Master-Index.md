AMI-001 — Aureus Master Index

Status: Living Draft

Authority: Repository Governance

Purpose

The Aureus Master Index (AMI) is the official navigation system for the Aureus repository.

Its purpose is to provide a single authoritative location from which every governing document, implementation document, blueprint, work order, policy, protocol, and operational standard can be located, understood, and traced.

The Master Index exists to ensure that no governing knowledge is lost, duplicated, misplaced, or forgotten as Aureus grows.

---

Objectives

The Master Index shall:

- Maintain a complete inventory of repository documents.
- Record each document's authority.
- Record each document's status.
- Record each document's owner.
- Record dependencies.
- Record supersession history.
- Record implementation status.
- Record review history.
- Provide navigation for humans and AI collaborators.
- Support repository audits.
- Support constitutional integrity.
- Preserve institutional knowledge.

---

Guiding Principles

The Master Index shall be:

- Accurate.
- Complete.
- Current.
- Traceable.
- Transparent.
- Searchable.
- Evidence-based.
- Easy to maintain.

---

Document Record Standard

Every indexed document shall contain, at minimum:

- Document Number
- Title
- Purpose
- GitHub Location
- Repository Folder
- Authority Level
- Status
- Version
- Author
- Approving Authority
- Date Created
- Last Updated
- Last Reviewed
- Next Review Date
- Related Documents
- Dependencies
- Supersedes
- Superseded By
- Implementation Status
- Review Notes

---

Authority Levels

Every document shall be classified according to its governing authority.

Examples include:

- Constitution
- Constitutional Amendment
- Organizational Standard
- Canon
- Blueprint
- Protocol
- Policy
- Work Order
- Technical Specification
- Implementation Guide
- Operational Procedure
- Supporting Documentation
- Archive

---

Status Values

Each document shall contain one status.

- Draft
- Living Draft
- Founder Review
- Approved
- Active
- Frozen
- Archived
- Superseded
- Deprecated

---

Repository Organization

The Master Index shall organize documents by category rather than discovery order.

Example categories include:

- Templates
- Archive

---

Traceability Requirements

Every document shall be traceable to:

- Higher governing authority.
- Related governing authority.
- Implementation.
- Tests, where applicable.
- Supporting documentation.

Broken traceability shall be considered a repository defect.

---

Supersession Rules

Documents shall never silently disappear.

When a document is replaced:

- the replacement shall be identified;
- the reason shall be recorded;
- historical references shall be preserved;
- repository history shall remain understandable.

---

AI Collaboration

AI collaborators shall use the Master Index as the primary navigation document before concluding that information is missing.

AI collaborators shall:

- search the index before recommending new documents;
- identify existing authority before suggesting changes;
- preserve repository organization;
- avoid duplicate concepts;
- report inconsistencies discovered during navigation.

---

Repository Health

The Master Index shall support ongoing detection of:

- duplicate documents;
- numbering conflicts;
- broken references;
- orphaned files;
- misplaced documents;
- outdated records;
- inconsistent terminology;
- superseded material requiring archival.

---

Quality Standards

The Master Index shall be reviewed before:

- Constitutional Freeze;
- major releases;
- production launch;
- governance revisions;
- repository restructuring.

---

Success Criteria

The Master Index is considered healthy when:

- every governing document is represented;
- every document has one authoritative location;
- every dependency is traceable;
- every supersession is documented;
- every active document has an owner and status;
- repository navigation can begin from the Master Index without ambiguity.

---

Continuous Stewardship

The Master Index is a living governance document.

It shall evolve with the institution while preserving the integrity, discoverability, and historical continuity of the Aureus repository.

Every significant addition to the repository should be reflected in the Master Index before that work is considered complete.
