# BRAND-006 — Typography System

Status: Draft for Founder Review

---

# Purpose

This document establishes the official typography system of Aureus.

Typography exists to communicate knowledge with clarity, consistency, accessibility, and professionalism.

The typography system should reinforce trust while reducing cognitive effort.

---

# Typography Philosophy

The typography of Aureus should feel:

- Clear
- Calm
- Modern
- Timeless
- Professional
- Human
- Intelligent
- Accessible

Typography should never compete with the message.

Instead, it should make every message easier to understand.

---

# Objectives

The typography system shall:

- Maximize readability.
- Improve comprehension.
- Reduce visual fatigue.
- Scale across every platform.
- Support accessibility.
- Create visual consistency.
- Reinforce the Aureus identity.

---

# Typography Hierarchy

The system shall define consistent styles for:

- Display Titles
- Page Titles
- Section Headings
- Subheadings
- Body Text
- Captions
- Labels
- Navigation
- Buttons
- Tables
- Code (when applicable)
- Quotes
- Footnotes

Every level should have a clear visual purpose.

---

# Readability Principles

Typography should prioritize:

- Comfortable line length
- Consistent spacing
- Clear hierarchy
- Appropriate font sizes
- Strong contrast
- Predictable rhythm

The reader should never struggle to locate important information.

---

# Accessibility

Typography shall:

- Meet or exceed WCAG AA accessibility standards.

AAA compliance should be preferred whenever practical.

The system should remain readable across mobile devices, desktop applications, presentations, printed materials, and documentation.

---

# Font Selection Principles

Official typefaces should be:

- Highly legible
- Professionally maintained
- Internationally supported
- Timeless
- Versatile
- Appropriate for long-form reading

Selection will consider:

- Screen performance
- Print performance
- Licensing
- Accessibility
- Multilingual support

---

# Consistency

The same typography system should be used across:

- Website
- Mobile applications
- Product interfaces
- Documentation
- Presentations
- Social media
- Marketing
- Printed materials
- Brand book
