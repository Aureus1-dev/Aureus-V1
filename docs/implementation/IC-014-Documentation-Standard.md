IC-014 — Documentation Standard

Canonical Designation: IC-014

Title: Documentation Standard

Status: Canonical

Authority: This document establishes the mandatory standards governing all documentation within the Aureus platform. It is subordinate to the OAS Series, the Product Constitution, the Architecture Decision Records (ADRs), the Implementation Decision Records (IDRs), and IC-001 through IC-013.

---

Article I — Purpose

The purpose of this Standard is to ensure that Aureus documentation remains accurate, complete, understandable, and synchronized with the software it describes.

Documentation is a permanent institutional asset and shall be maintained with the same discipline as source code.

---

Article II — Scope

This Standard applies to:

- Constitutional documents
- Product documentation
- Architecture documentation
- ADRs
- IDRs
- Work Orders
- API documentation
- Database documentation
- Deployment documentation
- Operational documentation
- User documentation
- Developer documentation
- Release documentation

---

Article III — Documentation Principles

Documentation shall be:

- Accurate
- Current
- Complete
- Traceable
- Understandable
- Version-controlled
- Consistent

Documentation shall describe reality rather than intention whenever practical.

---

Article IV — Canonical Source

Each subject shall have one canonical source.

Duplicate documentation describing the same authoritative information shall be avoided unless expressly required.

When duplicate reference material exists, the canonical source shall prevail.

---

Article V — Synchronization

Documentation shall be updated whenever implementation changes affect its accuracy.

A Work Order requiring documentation updates shall not be considered complete until those updates have been made.

---

Article VI — Organization

Documentation shall be organized according to the canonical repository structure.

Related documents shall reference one another where appropriate.

Cross-references shall remain current.

---

Article VII — Version History

Significant documentation changes should identify:

- Date
- Author or responsible steward
- Related Work Order(s)
- Related ADR(s)
- Related IDR(s)

Historical records shall be preserved.

---

Article VIII — Quality Standards

Documentation shall:

- Define terms consistently.
- Avoid ambiguity.
- Use professional language.
- Explain decisions where appropriate.
- Remain concise without sacrificing clarity.

---

Article IX — Review

Documentation associated with implementation shall be reviewed as part of the Code Review process.

Documentation quality is part of implementation quality.

---

Article X — Long-Term Stewardship

Documentation exists to preserve institutional knowledge across generations of contributors.

Every contributor shares responsibility for improving documentation whenever it becomes inaccurate, incomplete, or obsolete through approved processes.

The Documentation Standard shall evolve only through approved amendments that strengthen clarity, maintainability, and the long-term stewardship of the Aureus platform.
