# BRAND-017 — Iconography

Status: Living Draft

---

# Purpose

This document establishes the constitutional philosophy governing iconography throughout Aureus.

Icons are not decoration.

They are visual language that should help members understand, navigate, and interact with the institution more naturally.

Every icon should reduce cognitive effort while reinforcing trust, clarity, and human dignity.

---

# Philosophy

Icons exist to serve people.

They should communicate meaning quickly, consistently, and respectfully.

An icon should never require unnecessary interpretation.

Understanding should feel immediate and intuitive.

---

# Design Principles

Every Aureus icon should be:

• Simple

• Clear

• Timeless

• Friendly

• Accessible

• Consistent

• Recognizable

Icons should never feel:

• Aggressive

• Overly detailed

• Trend-driven

• Cartoonish

• Decorative for decoration's sake

---

# Emotional Objectives

The icon system should reinforce:

• Trust

• Calm

• Hope

• Clarity

• Hospitality

• Human dignity

Icons should support the emotional experience rather than distract from it.

---

# Consistency

Every icon should appear as though it belongs to the same visual family.

Consistency should exist across:

• Stroke weight

• Corner radius

• Scale

• Alignment

• Optical balance

• Visual rhythm

---

# Function Before Decoration

Icons exist to improve understanding.

If an icon creates confusion, additional decoration, or unnecessary visual complexity, it should be redesigned or removed.

---

# Accessibility

Icons shall never be the sole method of communicating important information.

Whenever practical:

• Icons should be paired with labels.

• Meaning should remain understandable without color.

• Icons should remain recognizable at small sizes.

• Accessibility technologies should not depend upon icon recognition alone.

---

# Illustration Relationship

Illustrations and icons serve different purposes.

Icons communicate function.

Illustrations communicate emotion and story.

These systems should complement one another while remaining visually distinct.

---

# Scalability

The icon system should function consistently across:

• Mobile applications

• Desktop applications

• Printed materials

• Educational resources

• Presentations

• Marketing materials

• Large-format displays

Icons should remain legible from very small interface elements to large signage.

---

# Evolution

The Aureus icon system may expand over time as new services, products, and experiences are introduced.

New icons should extend the established visual language rather than create competing styles.

---

# Living Draft

This document remains under active review while Aureus develops its complete visual language, conducts usability testing, and gathers long-term member feedback before canonization.
