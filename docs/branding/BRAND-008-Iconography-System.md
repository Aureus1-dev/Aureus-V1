# BRAND-008 — Iconography System

Status: Draft for Founder Review

---

# Purpose

This document establishes the official iconography system for Aureus.

Icons exist to improve understanding, reduce cognitive effort, and reinforce the Aureus visual identity.

They are functional communication tools, not decoration.

---

# Philosophy

Every icon should communicate:

- Clarity
- Simplicity
- Trust
- Consistency
- Accessibility

Icons should help people understand information faster.

They should never distract from the content they support.

---

# Objectives

The Aureus icon system shall:

- Improve usability.
- Reduce ambiguity.
- Increase recognition.
- Create consistency.
- Support accessibility.
- Scale across every platform.

---

# Design Principles

Icons should be:

- Simple
- Geometric
- Timeless
- Balanced
- Legible
- Consistent
- Human-centered

Every icon should feel like part of the same family.

---

# Visual Style

The official icon system should maintain consistent:

- Stroke weight
- Corner radius
- Proportions
- Alignment
- Perspective
- Visual balance
- Level of detail

Icons should avoid unnecessary ornamentation.

---

# Categories

The icon library will include standards for:

- Navigation
- Actions
- Communication
- People
- Organizations
- Opportunities
- Knowledge
- Resources
- Security
- Notifications
- Settings
- Finance
- Education
- Health
- Community
- AI
- Media
- Files
- Analytics
- Success Indicators

---

# Accessibility

Icons shall never be the sole method of communicating important information.

Whenever appropriate, icons should be paired with clear text labels.

Icons should remain recognizable at small sizes.

---

# Consistency

The same icon language shall be used across:

- Website
- Mobile applications
- Desktop applications
- Documentation
- Presentations
- Dashboards
- Marketing materials

---

# Future Expansion

Future versions of this document will define:

- Official icon library
- Stroke sizes
- Grid system
- Icon spacing
- Component usage
- Filled variants
- Outlined variants
- Animated icons
- Accessibility guidance

---

# Canonical Principle

Icons should simplify understanding.

They should never compete with the message they represent.

---

End of BRAND-008.
